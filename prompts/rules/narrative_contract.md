# 小说生成规则契约

本契约是 NovelAgent 默认长篇章节写作规则的人类可读版本。结构化来源为 `rules/default_narrative_rule_pack.json`。

## 章节连续性

- 必须接住上一章结尾：新章节开头必须承接上一章最后的地点、人物、动作或未完成事件，不得无解释跳转。
- 保持世界规则一致：世界规则、能力代价、空间限制和已建立的危险机制不能被随意改写；例外必须有明确铺垫。

## 人物一致性

- 保留上一场景人物：上一章末尾仍在场的人物不能在新章节开头无故消失；如果离场，必须交代行动、原因或代价。
- 保持人物状态一致：角色的伤势、目标、立场、知识范围和情绪状态不能无解释改变；变化必须来自本章可见事件。

## 空间连续性

- 保持上一场景地点连续：章节开头不得无解释突然换地点；如果需要转场，必须写清楚从上一地点到新地点的行动或原因。

## 冲突推进

- 推进当前冲突或伏笔：每章至少推进一个当前冲突、目标、威胁或伏笔，可以升级、揭示局部信息或制造新的选择代价。
- 章节结束必须改变故事状态：章节结束时必须产生可记录的状态变化，例如位置变化、关系变化、冲突升级、线索获得、代价产生或目标改变。

## 伏笔处理

- 避免提前回收伏笔：标记为未解决、不可回收或需要延后的伏笔，不得在没有铺垫和代价的情况下提前彻底解决。
- 重大揭示必须有铺垫：重大真相、能力、身份、背叛或世界设定揭示必须有前文铺垫，不得凭空出现解决剧情。

## 语言与风格

- 使用目标语言：章节必须使用项目指定目标语言；zh-CN 项目应输出简体中文连续小说正文。
- 保持视角稳定：章节应保持稳定叙事视角，不要无意义频繁切换人物内心或全知解释。

## 节奏与推进

- 避免重复和原地踏步：避免重复段落、水文、反复解释同一情绪或让人物长时间无有效行动。
- 章节长度合理：章节长度应足以完成一个有效场景推进，也不应长到失去章节边界。
- 场景必须有具体行动：每个主要场景都应包含具体动作、决策、发现、代价或状态变化，不能只写心理和解释。

## 输出格式

必须输出：

- 只输出小说正文
- 使用目标语言
- 延续当前章节状态
- 推进至少一个当前冲突或伏笔

## 禁止事项

- 不要输出分析
- 不要输出总结
- 不要输出 JSON
- 不要输出 Markdown 标题
- 不要解释你将如何写
